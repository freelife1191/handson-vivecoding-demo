# 08차시 프론트엔드 모듈 설정

---

1. 백엔드 연결 방식
```
프론트 개발을 독립적으로 완료하고 그 다음 백엔드에 연결하는 방식으로 개발하도록 @checklist.md 파일을 수정해줘.
```

2. 체크리스트 진행
```
@checklist.md 1.3 진행해줘.
```

3. 테스트 누락 체크리스트 보완
```
@design.md 의 테스트 전략이 @checklist.md 에 반영되었는 지 확인해주고 누락된 부분이 있으면 @checklist.md 를 수정해줘
```

4. 전체적인 체크리스트 보완
```
@design.md 의 내용이 @checklist.md 에 반영이 안된 다른 부분들이 있는지 확인해주고 있다면 반영해줘
```

5. 1.3 단계 진행 재확인
```
@checklist.md 진행된 1.3 단계에도 테스트 코드가 보이지 않는 것 같은데 단계가 누락됐다면 확인해서 @checklist.md 에 반영하고 진행해줘
```

6. 체크리스트 재검토
```
@requirements.md @design.md 문서를 참고해서
@checklist.md 누락되거나 잘못반영된 부분이 없는지 다시 한번 검토해줘
```

7. Vitest 필요성 검토
```
이제 내가 더 염려해야될 부분은 없겠지? vi test는 jest 와 병행해서 사용되지 않아? vi test 는 별로야?
```

8. Vitest 도입
```
vi test 도 병행해서 테스트를 진행하는게 좋다면 @requirements.md @design.md 문서에도 포함시켜주고 @checklist.md 에도 반영해줘
```

9. Vitest 도입 후 1.3 재진행 확인
```
1.3 을 다시 진행할필요있어?
```

## 참고사항

### 개발 방식
- **프론트엔드 우선 개발**: 프론트엔드를 완전히 독립적으로 구현한 후 백엔드와 연결
- **로컬 스토리지 기반**: 초기에는 로컬 스토리지를 사용하여 완전한 독립 동작 보장
- **TDD 적용**: 코어 비즈니스 로직(Todo 모델, 상태 관리, 스토리지 어댑터, 커스텀 훅)은 반드시 TDD 방식으로 구현
- **단계별 커밋**: 각 작업이 완료될 때마다 해당 체크리스트 항목을 체크하고 커밋

### 기술 스택
- **UI 프레임워크**: Mantine UI 컴포넌트 활용
- **언어**: 모든 코드는 TypeScript로 작성하며, 적절한 타입 정의 포함
- **상태 관리**: React Context API + useReducer 패턴
- **스토리지**: 로컬 스토리지 → API 전환 방식

### 품질 기준
- **접근성**: WCAG 2.1 AA 수준 준수
- **반응형**: 모바일 우선 반응형 디자인
- **성능**: React.memo, useMemo, useCallback 등 최적화 기법 적용
- **테스트**: 단위 테스트, 통합 테스트, E2E 테스트 완비

### 개발 흐름
1. **1-6단계**: 프론트엔드 완전 독립 개발 (로컬 스토리지 기반)
2. **7-8단계**: 백엔드 API 개발 (TDD)
3. **9단계**: 프론트엔드-백엔드 통합 (스토리지 전환)
4. **10단계**: 인프라 및 배포

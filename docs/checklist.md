## 참고사항

### 개발 방식
- **프론트엔드 우선 개발**: 프론트엔드를 완전히 독립적으로 구현한 후 백엔드와 연결
- **로컬 스토리지 기반**: 초기에는 로컬 스토리지를 사용하여 완전한 독립 동작 보장
- **TDD 적용**: 코어 비즈니스 로직(Todo 모델, 상태 관리, 스토리지 어댑터, 커스텀 훅)은 반드시 TDD 방식으로 구현
  - **TDD 사이클**: 실패하는 테스트 작성 → 최소한의 코드 작성 → 리팩토링 → 반복
  - **백엔드**: 모든 구현을 TDD로 진행
  - **프론트엔드**: UI 구현 시에는 실행 코드를 먼저 작성하고, 코어 비즈니스 로직 구현 시에만 TDD 진행
- **단계별 커밋**: 각 작업이 완료될 때마다 해당 체크리스트 항목을 체크하고 커밋

### 기술 스택
- **UI 프레임워크**: Mantine UI 컴포넌트 활용
- **언어**: 모든 코드는 TypeScript로 작성하며, 적절한 타입 정의 포함
- **상태 관리**: React Context API + useReducer 패턴
- **스토리지**: 로컬 스토리지 → API 전환 방식
- **테스트**: Vitest (Jest 호환 API) + React Testing Library
- **빌드 도구**: Vite (개발 서버 + 테스트 통합)

### 품질 기준
- **접근성**: WCAG 2.1 AA 수준 준수
- **반응형**: 모바일 우선 반응형 디자인
- **성능**: 페이지 로드 시간 < 2초 목표 (requirements.md 명세)
- **브라우저 호환성**: 최신 버전의 Chrome, Firefox, Safari, Edge 지원
- **테스트**: 단위 테스트, 통합 테스트, E2E 테스트 완비

### UI/UX 디자인 원칙
- **모바일 우선 디자인**: 모든 화면 크기에서 최적화된 경험
- **직관적인 사용자 인터페이스**: 사용자가 쉽게 이해할 수 있는 UI
- **Mantine 디자인 시스템**: 일관된 컬러/타이포그래피/컴포넌트 스타일 유지
- **반응형 레이아웃**: Grid, Flex 등 Mantine 컴포넌트 활용
- **고급 컴포넌트 활용**: Notification, Modal 등 Mantine 컴포넌트 적극 사용

### Git Hook 자동화
- **Pre-commit Hook**: 루트 `.husky/pre-commit`에서 모노레포 전체 관리
- **실행 조건**: frontend 디렉토리 변경 시에만 실행
- **자동 실행 순서**: Lint Fix → Build → Test
- **코드 품질 보장**: 커밋 전 자동 검증으로 품질 저하 방지

### 개발 흐름 (requirements.md 마일스톤 우선순위 반영)
1. **1-6단계**: 프론트엔드 완전 독립 개발 (로컬 스토리지 기반)
   - 기본 TODO CRUD 기능 구현 (1순위)
   - UI/UX 개선 (2순위)
   - 필터링 및 정렬 기능 구현 (3순위)
2. **7-8단계**: 백엔드 API 개발 (TDD)
   - 데이터 관리 기능 구현 (4순위)
3. **9단계**: 프론트엔드-백엔드 통합 (스토리지 전환)
4. **10단계**: 인프라 및 배포
   - 추가 기능 구현 (5순위, 사용자 피드백 기반)
